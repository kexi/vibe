## パッケージマネージャー

<Tabs>
  <TabItem label="Homebrew (macOS)">
    ```bash frame="terminal"
    brew install kexi/tap/vibe
    ```
  </TabItem>
  <TabItem label="npm (Node.js)">
    ```bash frame="terminal"
    # グローバルインストール
    npm install -g @kexi/vibe

    # またはnpxで直接実行
    npx @kexi/vibe start feat/my-feature
    ```

    :::note
    `@kexi/vibe` は薄いランチャーです。インストール時に `optionalDependencies` 経由でホストに合うプラットフォーム別パッケージ（`@kexi/vibe-<platform>-<arch>`）を取得し、それに含まれる Rust バイナリを実行します。Copy-on-Write ファイルクローニング（macOS APFS、Linux Btrfs/XFS）はその Rust バイナリに組み込まれています。
    :::

  </TabItem>
  <TabItem label="Bun">
    ```bash frame="terminal"
    # グローバルインストール
    bun add -g @kexi/vibe

    # またはbunxで直接実行
    bunx @kexi/vibe start feat/my-feature
    ```

    :::note
    Bun 1.2.0以降が必要です。`@kexi/vibe` は薄いランチャーです。インストール時に `optionalDependencies` 経由でホストに合うプラットフォーム別パッケージ（`@kexi/vibe-<platform>-<arch>`）を取得し、それに含まれる Rust バイナリを実行します。Copy-on-Write ファイルクローニング（macOS APFS、Linux Btrfs/XFS）はその Rust バイナリに組み込まれています。
    :::

  </TabItem>
  <TabItem label="mise">
    `.mise.toml`に追加：

    ```toml
    [plugins]
    vibe = "https://github.com/kexi/mise-vibe"

    [tools]
    vibe = "latest"
    ```

    その後、インストール：

    ```bash frame="terminal"
    mise install
    ```

    :::tip
    mise hooksで`vibe shell-setup`を使うと[手動シェル設定](/ja/setup/)をスキップできます。
    詳しくは[自動セットアップ](/ja/setup/#miseを使った自動セットアップ)をご覧ください。
    :::

  </TabItem>
  <TabItem label="Nix">
    ```bash frame="terminal"
    # 直接実行（一時的）
    nix run github:kexi/vibe -- start feat/my-feature

    # 永続インストール
    nix profile install github:kexi/vibe
    ```

    NixOSまたはHome Managerの場合、flake inputとして追加：

    ```nix
    {
      inputs.vibe.url = "github:kexi/vibe";

      # 設定で使用：
      # environment.systemPackages = [ inputs.vibe.packages.${system}.default ];
    }
    ```

    :::note
    デフォルトパッケージ（`packages.default` / `.#fromSource`）は再現性のため
    ソースからビルドされます。ビルド済みのリリースバイナリ（コンパイル不要、
    SHA-256ハッシュで検証）を使いたい場合は `packages.binary` を指定します：

    ```bash frame="terminal"
    nix run github:kexi/vibe#binary -- start feat/my-feature
    ```
    :::

  </TabItem>
</Tabs>

## Linux

:::note
WSL2ユーザーは、使用しているディストリビューションに応じて以下のLinuxインストール方法を使用できます。
:::

<Tabs>
  <TabItem label="Ubuntu/Debian">
    ```bash frame="terminal"
    # x64
    curl -LO https://github.com/kexi/vibe/releases/latest/download/vibe_amd64.deb
    sudo apt install ./vibe_amd64.deb

    # ARM64
    curl -LO https://github.com/kexi/vibe/releases/latest/download/vibe_arm64.deb
    sudo apt install ./vibe_arm64.deb

    # アンインストール
    sudo apt remove vibe
    ```

  </TabItem>
  <TabItem label="その他のディストリビューション">
    ```bash frame="terminal"
    # x64
    curl -L https://github.com/kexi/vibe/releases/latest/download/vibe-linux-x64 -o vibe
    chmod +x vibe
    sudo mv vibe /usr/local/bin/

    # ARM64
    curl -L https://github.com/kexi/vibe/releases/latest/download/vibe-linux-arm64 -o vibe
    chmod +x vibe
    sudo mv vibe /usr/local/bin/
    ```

  </TabItem>
</Tabs>

## Windows

:::caution
Windows向けバイナリ配布は再構築中です。現在のリリースには `vibe-windows-x64.exe` アセットが含まれず、`@kexi/vibe-win32-x64` パッケージもまだ存在しません。そのため Windows で npm から `@kexi/vibe` をインストールしてもランチャーが実行するバイナリを見つけられず、明示的なエラーで終了します。

ネイティブな Windows ビルドが復活するまでは、[WSL2](https://learn.microsoft.com/ja-jp/windows/wsl/) 上で上記の [Linux](#linux) 手順を使うか、Rust ツールチェーンで[ソースからビルド](#手動ビルド)してください。
:::

## 手動ビルド

[Rust ツールチェーン](https://rustup.rs/)がインストールされていれば、ソースからバイナリをビルドできます：

```bash frame="terminal"
git clone https://github.com/kexi/vibe.git
cd vibe
cargo build --manifest-path rust/Cargo.toml -p vibe --release
# バイナリは rust/target/release/vibe に出力されます
```
